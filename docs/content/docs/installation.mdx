---
title: Installation
description: Install Opengram using Docker, the install script, or manually from source.
---

Opengram can be installed in three ways depending on your environment and preferences.

## Prerequisites

- **Node.js 20+** (the install script will install it for you if needed)
- **Linux** for the systemd service (Docker works on any platform)

## Docker (quickest)

The fastest way to get Opengram running is with Docker:

```bash
docker run -d \
  --name opengram \
  -p 3000:3000 \
  -v opengram_data:/opt/opengram/data \
  opengram/web:latest
```

This starts Opengram on port 3000 with a persistent volume for the database and uploaded files.

## Direct install (Linux systemd)

For a permanent installation on a Linux server, use the install script from the repository root:

```bash
sudo ./install.sh
```

The script will:

1. Install Node.js 20 if it is not already present.
2. Build the application.
3. Create the directory structure under `/opt/opengram/`.
4. Set up a systemd service for automatic startup.

After installation, run `opengram init` to walk through initial configuration.

## Manual (development)

Requires **Node.js 20+**.

```bash
git clone https://github.com/nichochar/opengram.git
cd opengram
npm install
npm run build
npm start
```

## Directory layout

After installation, the filesystem is organized as follows:

```
/opt/opengram/
  web/              # Application files
  data/
    opengram.db     # SQLite database
    uploads/        # Uploaded files
  config/
    opengram.config.json
    opengram.env
```

- **`web/`** contains the built application code.
- **`data/`** holds all persistent state -- the SQLite database and uploaded files. Back up this directory to preserve everything.
- **`config/`** contains the JSON configuration file and environment variables. See [Configuration](/docs/configuration) for details.

## Verify the installation

After starting Opengram, verify it is running:

```bash
curl -fsS http://localhost:3000/api/v1/health
```

You should see `{"status":"ok"}`.

## Database migrations

Database migrations run automatically on first startup. Both the Docker entrypoint and the install script handle this, so no manual migration step is needed. On subsequent upgrades, new migrations are applied automatically when the server starts.
