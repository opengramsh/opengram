Opengram supports tagging chats for organization. Tags let users categorize conversations and quickly filter the chat list to find what they need.

## Adding Tags

Tags can be added to any chat from the chat header or chat settings. Type a tag name to create a new tag or select from existing suggestions.

### Autocomplete

When adding a tag, Opengram suggests tags that already exist across your chats. This helps keep tag names consistent and avoids duplicates from typos or variations.

The suggestion list is fetched from the API:

```
GET /api/v1/tags/suggestions?query=proj
```

## Filtering by Tag

In the chat list, users can filter by one or more tags to narrow down the visible conversations. Only chats that have the selected tag(s) are shown.

This is useful for organizing chats by project, topic, priority, or any other grouping that makes sense for your workflow.

## Managing Tags

Tags are stored per chat and can be added or removed at any time. To update a chat's tags, use the [Update Chat](/docs/api-reference/updateChat) endpoint:

```
PATCH /api/v1/chats/:chatId
```

```json
{
  "tags": ["project-alpha", "urgent"]
}
```

Removing a tag from all chats effectively deletes it -- there is no separate tag management interface. Tags exist as long as at least one chat uses them.

## API Reference

- [Update chat tags](/docs/api-reference/updateChat) -- `PATCH /api/v1/chats/:chatId`
- [Tag suggestions](/docs/api-reference/listTagSuggestions) -- `GET /api/v1/tags/suggestions`
