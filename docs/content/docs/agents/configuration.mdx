---
title: Agent Configuration
description: Define agents and models in the Opengram configuration file so the UI and dispatch system know how to route messages.
---

Agents and models are defined in `opengram.config.json`. Agents represent the AI personas that respond to users, while models represent the LLMs those agents can use. The UI uses this configuration to display agent names, avatars, and model selectors.

## Defining agents

Add an `agents` array to your config file. Each agent object supports the following fields:

| Field | Required | Description |
| --- | --- | --- |
| `id` | Yes | Unique identifier referenced when creating messages via the API. |
| `name` | Yes | Display name shown in the chat UI. |
| `description` | No | Short text shown in the agent picker. |
| `avatarUrl` | No | URL to an avatar image. Can be an absolute URL or a path relative to the base URL. |
| `defaultModelId` | No | The model this agent uses by default. Must match an `id` in the `models` array. |

## Defining models

Add a `models` array to your config file. Each model object supports:

| Field | Required | Description |
| --- | --- | --- |
| `id` | Yes | Unique identifier referenced in API calls and agent defaults. |
| `name` | Yes | Display name shown in the model selector. |
| `description` | No | Short text describing the model's capabilities. |
| `metadata` | No | Arbitrary key-value object passed through to your agent at dispatch time. |

## Default model for new chats

Set `defaultModelIdForNewChats` at the top level of the config to control which model is selected when a user creates a new chat. This must match an `id` from the `models` array.

## Example configuration

```json title="opengram.config.json"
{
  "agents": [
    {
      "id": "assistant",
      "name": "Assistant",
      "description": "General-purpose assistant",
      "avatarUrl": "/images/assistant.png",
      "defaultModelId": "gpt-4o"
    },
    {
      "id": "coder",
      "name": "Code Helper",
      "description": "Specialized in programming tasks",
      "defaultModelId": "claude-sonnet"
    }
  ],
  "models": [
    {
      "id": "gpt-4o",
      "name": "GPT-4o",
      "description": "Fast and capable general model"
    },
    {
      "id": "claude-sonnet",
      "name": "Claude Sonnet",
      "description": "Strong at reasoning and code"
    },
    {
      "id": "llama-3",
      "name": "Llama 3 70B",
      "description": "Open-weight model for self-hosted inference",
      "metadata": {
        "endpoint": "http://localhost:11434"
      }
    }
  ],
  "defaultModelIdForNewChats": "gpt-4o"
}
```

## How IDs are used

When your agent creates a message through the API, it passes its `agentId` and the `modelId` to identify itself. These must match the IDs defined here. The dispatch system also includes these IDs in the payload so your worker knows which agent and model should handle a given batch. See [Dispatch System](/docs/agents/dispatch) for details.
