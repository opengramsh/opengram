title: Configuration Reference
description: Complete reference for opengram.config.json
---

This is the full schema of `opengram.config.json`. All fields are optional and fall back to their default values when omitted.

## Top-Level Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `appName` | string | `"OpenGram"` | Display name shown in the UI and PWA manifest |
| `maxUploadBytes` | number | `50000000` | Maximum file upload size in bytes (50 MB) |
| `allowedMimeTypes` | string[] | `["*/*"]` | Allowed MIME types for file uploads. Supports wildcards like `"image/*"` |
| `titleMaxChars` | number | `48` | Maximum character length for chat titles |
| `defaultModelIdForNewChats` | string \| undefined | `undefined` | Model ID to preselect when creating a new chat. Must match an ID in the models array |
| `agents` | AgentConfig[] | `[]` | Agent definitions (see below) |
| `models` | ModelConfig[] | `[]` | Model definitions (see below) |

### Validation

- At least **1 agent** and **1 model** must be configured.
- If `defaultModelIdForNewChats` is set, it must reference a model ID in the `models` array.

## `agents`

Each entry in the `agents` array defines an agent that can handle conversations.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `id` | string | (required) | Unique identifier for the agent |
| `name` | string | (required) | Display name shown in the UI |
| `description` | string | `""` | Short description of the agent |
| `avatarUrl` | string \| undefined | `undefined` | URL to the agent's avatar image |
| `defaultModelId` | string \| undefined | `undefined` | Default model for this agent. Must reference a model in the `models` array |

## `models`

Each entry in the `models` array defines a model that agents can use.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `id` | string | (required) | Unique identifier for the model |
| `name` | string | (required) | Display name shown in the model selector |
| `description` | string | `""` | Short description of the model |
| `metadata` | Record\<string, string\> | `{}` | Arbitrary key-value pairs for your agent to consume (e.g. endpoint URL, context window size) |

## `push`

Push notification settings. See [Push Notifications](/docs/features/push-notifications) for setup details.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `push.enabled` | boolean | `false` | Enable web push notifications |
| `push.vapidPublicKey` | string | `""` | VAPID public key for push subscriptions |
| `push.vapidPrivateKey` | string | `""` | VAPID private key for sending push messages |
| `push.subject` | string | `""` | VAPID subject (must be an `https://` URL or `mailto:` address when push is enabled) |

### Validation

- When `push.enabled` is `true`, all three key/subject fields are required.
- The `subject` field cannot contain `localhost` -- Apple Web Push will reject it.

## `security`

Authentication and access control settings.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `security.instanceSecretEnabled` | boolean | `false` | Require a bearer token for write endpoints |
| `security.instanceSecret` | string | `""` | The bearer token value. Generated by `opengram init` with an `og_` prefix followed by a random base64url string |
| `security.readEndpointsRequireInstanceSecret` | boolean | `false` | Also require the token on read-only (GET) endpoints |

### Validation

- When `instanceSecretEnabled` is `true`, `instanceSecret` must be a non-empty string.

## `server`

Server runtime settings.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `server.port` | number | `3000` | HTTP port the server listens on (1--65535) |
| `server.publicBaseUrl` | string | `"http://localhost:3000"` | Public base URL used for push notifications, webhooks, and generated links |
| `server.corsOrigins` | string[] | `[]` | Allowed CORS origins. Empty means all origins are allowed. Each entry must be a valid HTTP or HTTPS origin (no path, query, or hash) |
| `server.streamTimeoutSeconds` | number | `60` | Maximum seconds a streaming message can remain in the `streaming` state before auto-cancellation |
| `server.idempotencyTtlSeconds` | number | `86400` | How long (in seconds) idempotency keys are cached. Must be a positive integer. Default is 24 hours |

## `server.dispatch`

Controls how chat messages are batched, queued, and delivered to agent workers. See [Dispatch System](/docs/agents/dispatch) for a full walkthrough.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `server.dispatch.mode` | string | `"batched_sequential"` | Dispatch mode: `"immediate"`, `"sequential"`, or `"batched_sequential"` |
| `server.dispatch.batchDebounceMs` | number | `600` | Milliseconds to wait after the last message before sealing a batch |
| `server.dispatch.typingGraceMs` | number | `2000` | Extra grace period (ms) if the user is still typing |
| `server.dispatch.maxBatchWaitMs` | number | `30000` | Maximum milliseconds before forcing a batch to dispatch |
| `server.dispatch.schedulerTickMs` | number | `500` | Interval (ms) for the batch scheduler's internal tick |
| `server.dispatch.leaseMs` | number | `30000` | Lease duration (ms) for claimed dispatches. Workers must heartbeat before this expires |
| `server.dispatch.heartbeatIntervalMs` | number | `5000` | Recommended heartbeat interval (ms) for workers processing a dispatch |
| `server.dispatch.claimWaitMs` | number | `10000` | How long a claim request long-polls when no work is available |
| `server.dispatch.retryBaseMs` | number | `500` | Base delay (ms) for exponential backoff on failed dispatches |
| `server.dispatch.retryMaxMs` | number | `30000` | Maximum delay (ms) for exponential backoff |
| `server.dispatch.maxAttempts` | number | `8` | Maximum number of attempts before a dispatch is permanently failed. Minimum: 1 |

### `server.dispatch.execution`

Controls autoscaling behavior for concurrent workers.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `server.dispatch.execution.autoscaleEnabled` | boolean | `true` | Whether autoscaling is active |
| `server.dispatch.execution.minConcurrency` | number | `2` | Minimum number of concurrent workers. Minimum: 1 |
| `server.dispatch.execution.maxConcurrency` | number | `10` | Maximum number of concurrent workers. Must be â‰Ą `minConcurrency` |
| `server.dispatch.execution.scaleCooldownMs` | number | `5000` | Cooldown period (ms) before scaling down |

### `server.dispatch.claim`

Controls the claim-many endpoint behavior.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `server.dispatch.claim.claimManyLimit` | number | `10` | Default number of batches returned by `claim-many`. Hard-capped at 50 |

## `hooks`

Webhook configuration for event-driven integrations. The `hooks` field is an array of webhook endpoint definitions. See [Webhooks](/docs/agents/webhooks) for usage details.

Each entry in the array supports:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `url` | string | (required) | The URL Opengram will POST to. Must be a valid URL |
| `events` | string[] | (required) | Array of event names to subscribe to (e.g. `["message.created", "request.resolved"]`) |
| `headers` | Record\<string, string\> | `{}` | Extra headers to include in every delivery request |
| `signingSecret` | string \| undefined | `undefined` | Secret used to compute the HMAC-SHA256 signature sent in `X-OpenGram-Signature` |
| `timeoutMs` | number | `5000` | Request timeout in milliseconds |
| `maxRetries` | number | `3` | Maximum number of retry attempts on failure |

### Example

```json
{
  "hooks": [
    {
      "url": "https://api.example.com/webhook",
      "events": ["message.created", "request.resolved", "request.cancelled"],
      "signingSecret": "whsec_your_secret",
      "timeoutMs": 10000,
      "maxRetries": 5
    }
  ]
}
```

## `autoRename`

Automatic chat renaming using an LLM. See [Auto-Rename](/docs/features/auto-rename) for provider-specific details.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `autoRename.enabled` | boolean | `false` | Enable automatic chat renaming after the first exchange |
| `autoRename.provider` | string | `""` | LLM provider: `"anthropic"`, `"openai"`, `"google"`, `"xai"`, or `"openrouter"` |
| `autoRename.modelId` | string | `""` | Model ID to use for generating titles (e.g. `"claude-haiku-4-5"`) |
| `autoRename.apiKey` | string \| undefined | `undefined` | API key for the provider. Optional if the corresponding environment variable is set (e.g. `ANTHROPIC_API_KEY`) |

### Validation

- When `enabled` is `true`, both `provider` and `modelId` are required.
- `provider` must be one of: `anthropic`, `openai`, `google`, `xai`, `openrouter`.
